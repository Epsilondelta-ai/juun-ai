---
name: git-style
description: Git workflow
---

1. Create a branch — branch from `main` with a name that clearly describes the work.
2. Commit — keep units of work small and write meaningful commit messages.
3. Create a PR — clearly explain the changes and intent.
4. Review & feedback — apply reviewer feedback and revise.
5. Merge to main — merge into `main` after review is complete.
